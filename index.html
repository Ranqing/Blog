处理字符串是动态规划经典题目类型之一，包括：最长递增子序列/子串；最长公共子序列/子串；最长回文子序列/子串；其中子串要求元素是连续的，而子序列不需要。这篇文章讲先对动态规划进行回顾，然后分别对以上问题进行分析，给出代码。

动态规划算法

总的来说，动态规划是一种解决问题的方法。

用小学数学进行解释：如果我们知道1+1+1+1+1 = 5，那么可以很快计算出1+1+1+1+1+1 = 6，这是因为之前已经计算过前5项的值，即出现了重复子问题。这也类似于在证明中常用的数学归纳法，假设在问题规模为N时结论成立，只需要证明该结论为N+1时结论也成立，则该结论即是普适的。另外知乎上还有高赞回答提到可以用数列中的通项公式来理解动态规划。

以上这些理解方式的共同点在于，要求解大规模问题（N+1）时，先去求解小规模的该问题（N+1），即找到重复子问题，然后记录小规模的子问题的值，最后快速得到大规模的该问题的结果。

Two-view Geometry

对极几何 / Epipolar Geometry

本文学习Two-View Geometry，假设两个相机可以看到空间同一个点P，在左右相机的投影点分别为p，q，它们之间存在一些几何关系，称之为Epipolar Gemotry。

几何性质

  1. 极平面 / Epipolar Plane
    像素点p，q与各自相机中心的连线: (k_p)^-1·p, (k_q)^-1·q 和相机中心连线（基线：baseline）组成的平面；由两个相机中心和三维点组成的平面。

  2. 极线 / Epipolar Line
    Epipolar Plane和图像平面的交线，互为三维点与对应的相机中心的光线的投影。p点在第二张图的对应位置只能在该图上的epipolar line上，反之亦然。

  3. 极点 / Epipole:
    两个相机中心的连线与图像平面的交点。还可以理解成为：一个同时可见的三维点与两个相机中心组成一个极平面，那么可以想象所有的极平面是绕着基线在旋转。这些极平面与图像相交得到的极线的交点即为epipole，即所有极线有唯一交点。

代数表示

  1. Essential Matrix

相机内参已知，那么由像素点p，q可以得到两个三维向量的方向： (k_p)^-1·p 和(k_q)^-1·q，此时这两个向量分别位于各自相机坐标系下，为了简写，直接记做p，q。
当两个相机之间的旋转R和平移T已知，先把第二个相机中的三维向量根据旋转矩阵R变换到第一个相机的坐标系中，为R^T·q，再根据Epipolar Geometry中的极平面的共面约束 -> p, t, R^T*q三个向量共面（在第一个相机坐标系下）；
共面约束在代数中表示为： a,b,c三个向量共面，那么其混合积Mix Product - cross-product(b,c).dot(a)为0。另，三个向量的混合集也可以表示这三个向量所组成平行六面体的体积。
由此，有代数方程：p·(txR^Tq) = 0，先把cross-product变换为矩阵计算[t]_x = []，那么改代数方程可以写为 p^T·[t]_xR^Tq) = 0，再整体进行一次transpose，得到：

其中的矩阵R[t]_x即为Essential Matrix E，这里的p，q分别是相机坐标系由内参和像素点得到的三维向量。
以上，Essential Matrix的物理意义可以简单的概括为：两个相机内参已知，p，q是相机坐标系下的三维向量，满足等式：pEq = 0，E=R[t]_x 即若干个三维点在双目中可见，那么这些三维点在各自相机坐标系下的坐标应该满足的代数关系，E可以被分解为两个相机之间的相对外参：R，t。

另外epipole的求解：假设e1是第二个相机中心c2在第一个相机的图像平面的投影点，e2是第一个相机中心c1在第二个相机平面的投影点。由于所有的极线都通过epipole，那么对于相机1的任意的一个像素点p，那么其极线Fp都通过epipole e2，意味着 (e2^T)Fp=0，对于任何一个p都成立，那么(e2^T)F=0，反之有F(e1)=0成立。

F的大小为3x3，初始自由度为9，由于方程定义在摄影几何，是equal up to scale，因此自由度实际为8。另外由于epipole的存在，那么rank(F)=2， 因此F的自由度为7。

How to build a blog

%—
%title: 如何在github.io的子目录下搭建博客
%—

由于之前已经用了github.io这个repo来作为个人主页，因此想把个人博客作为该主页的二级目录，试了很多方法最后终于成功了。
这篇文章是可行方法的具体步骤，同时也作为该博客的测试篇！

准备

搭建博客之前，我已经在github创建了username.github.io，且在aliyun上买好了域名做好了映射。
以上都完成好之后，在本地安装node.js 和 hexo。

安装命令(mac)

需要去node.js官网下载pkg安装软件，安装好进行检查。

安装node.js无误之后再利用npm安装hexo

可能会遇到一些问题：

安装完成之后使用hexo -v进行检查

建立本地的hexo blog

建立Github Repo，并且修改hexo blog下的_config.yml

直接在Github上New一个Repo，名称叫做所需网址的二级目录名称（e.g. blog)
修改_config.yml

执行部署

打开github上blog的setting界面，Github Pages勾选master branch。
此时输入网址 http://www.username.com/blog 就可以访问了。
所有的文章的位置在my-blog/source/_posts 目录下，且使用markdown语法。
